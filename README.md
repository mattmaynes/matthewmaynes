# matthewmaynes.com

Personal website for Matthew Maynes - a portfolio, resume, blog, and contact page. It is a small,
statically generated Next.js site with a working contact form and a Markdown-driven blog, served as
a self-contained Node standalone build.

## What is here

- **Home, About, Projects** - short, hand-written pages introducing the site.
- **Resume** - rendered from a scrubbed source (`src/lib/resume.ts`) and also downloadable as a PDF
  (`public/resume.pdf`, generated by `npm run resume:pdf`).
- **Blog** - MDX posts under `content/blog/`, with tag filtering, search, reading-time estimates,
  cover images, and an RSS 2.0 feed at `/blog/feed.xml`.
- **Contact** - a real form that emails through [Resend]; the destination address is a server-only
  secret and never ships to the browser.

## Stack

- **Next.js** (App Router) + **TypeScript**, built as a **Node standalone** output
  (`output: 'standalone'`).
- **Tailwind CSS v4** (CSS-first) on the **Harbor** theme - `@rogueoak/roots` design tokens with a
  site override in `src/styles/theme-harbor.css`.
- **Canopy** (`@rogueoak/canopy`) UI components (Button, Card, Badge, FormField, Input, Textarea).
- **MDX** blog content compiled with `next-mdx-remote`.
- **PostHog** for analytics, session replay, and error tracking (via a same-origin `/ingest` proxy).
- Self-hosted fonts via `@fontsource-variable/figtree` (UI/body) and
  `@fontsource-variable/geist-mono` (code).

## Local development

```bash
npm install
npm run dev
```

Visit http://localhost:3000. Pages: `/`, `/about`, `/resume`, `/projects`, `/blog`,
`/blog/<slug>`, `/contact`.

The contact form works without any configuration - if `RESEND_API_KEY` is unset it simply skips the
send. See [Configuration](#configuration) to wire up real delivery locally.

### Theme

Light/dark is driven by Roots' `.dark` class on `<html>`. It defaults to your OS setting
(`prefers-color-scheme`); the header toggle overrides it and the choice is persisted in
`localStorage` (key `theme`), winning over the system setting thereafter. A pre-paint inline script
applies the resolved theme before first paint, so there is no flash of the wrong theme.

## Production build

```bash
npm run build
node .next/standalone/server.js   # serves on PORT (default 3000), HOSTNAME 0.0.0.0
```

The standalone build emits `.next/standalone/server.js`. The runtime also needs `.next/static` and
`public/` copied alongside it (the Dockerfile does this).

## Docker

Multi-stage build (deps -> build -> runtime on `node:24-alpine`) serving the standalone output.

```bash
docker compose up --build
```

The site is served on http://localhost:3000. Equivalent manual commands:

```bash
docker build -t mm-site .
docker run --rm -p 3000:3000 mm-site
```

Production deployment (a Caddy reverse proxy in front of the site container) lives under
`deploy/docker/`.

## Configuration

Environment variables (see `.env.example` - copy to `.env.local` for local overrides):

| Variable                   | Purpose                                                             |
| -------------------------- | ------------------------------------------------------------------- |
| `SITE_URL`                 | Canonical origin for metadata and absolute URLs.                    |
| `NODE_ENV`                 | `development` \| `production` \| `test`.                            |
| `RESEND_API_KEY`           | Resend API key for contact-form delivery (server-only secret).      |
| `CONTACT_TO_EMAIL`         | Where contact messages are delivered (the private inbox).           |
| `CONTACT_FROM_EMAIL`       | Verified-domain sender address.                                     |
| `NEXT_PUBLIC_POSTHOG_KEY`  | Publishable PostHog project key (inlined at build; not a secret).   |
| `NEXT_PUBLIC_POSTHOG_HOST` | Client ingest host; defaults to the `/ingest` proxy.                |
| `POSTHOG_SERVER_CAPTURE`   | Set to `1` in production to enable server-side error reporting.     |

No secrets live in the repo. `.env*` is git-ignored except `.env.example`; the contact and PostHog
server values are never prefixed `NEXT_PUBLIC_`, so they stay out of the browser bundle.

## Notes

- This is a **public** repository. No real email, phone, or street address appears anywhere; the
  location is given only as "Ontario, Canada". The private resume master lives in the git-ignored
  `context/`; only its scrubbed derivative is tracked and rendered.
- Blog posts are prose plus a single `<PostImage>` component and ship via reviewed pull requests;
  see `CLAUDE.md` for the content and contribution rules.
