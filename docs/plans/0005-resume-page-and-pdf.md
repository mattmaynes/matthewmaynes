# Plan 0005 - Resume page + in-sync PDF download

Source: `docs/specs/0005-resume-page-and-pdf.md`.

## Steps

1. **`src/lib/resume.ts`** - tracked, typed, scrubbed resume data ported from `context/resume.md`
   minus the Contact block. No phone/email/street/postal; region only ("Ontario, Canada"). Convert
   all en-dashes to ASCII " - ". Shape: `summary`, `leadership[{lead, body}]`, `skills[]`,
   `tools[{label, items}]`, `work[{title, company, location, period, bullets[]}]`,
   `certifications[{date, name}]`, `education[{degree, school, location, period}]`.

2. **`src/app/resume/page.tsx`** - drop `PagePlaceholder`; render the data as real sections with
   Harbor tokens (`max-w-[1200px]`, `text-h1/h2/h3`, `text-body`, `Badge`). Header block: headshot
   (`priority`), name, title, region, LinkedIn + GitHub links (public, allowed), and a
   `Download PDF` `Button asChild` -> `<a href="/resume.pdf">` marked `print:hidden`. Work entries
   wrap in `break-inside-avoid`. Stays a Server Component.

3. **Print styles** - in `src/styles/theme-harbor.css`, a `@media print` block that (a) re-maps the
   semantic vars to the light palette under `:root, .dark` so the PDF is always light-on-white,
   (b) sets `@page { size: Letter; margin: 0.5in }`, (c) `print-color-adjust: exact` so badge
   colors render. Hide chrome via Tailwind `print:hidden` on the Header root, Footer root, the
   theme toggle, and the download button.

4. **`scripts/generate-resume-pdf.mjs`** - two modes:
   - default (generate): hash the resume **source inputs** (`src/lib/resume.ts`,
     `src/app/resume/page.tsx`, `src/styles/theme-harbor.css`); if `public/resume.pdf` +
     `public/resume.pdf.hash` already match and not `--force`, no-op. Else: ensure a standalone
     build, assemble + boot `server.js` on an ephemeral port, run headless Chrome
     (`--headless=new --no-pdf-header-footer --print-to-pdf --virtual-time-budget`), write
     `public/resume.pdf` and the hash sidecar. Locate Chrome via `CHROME_PATH` then common paths.
   - `--check`: recompute the source hash and compare to the committed `public/resume.pdf.hash`;
     exit nonzero on mismatch with "run npm run resume:pdf and commit". **No browser needed**, so
     CI runs only this - Chrome never enters CI or Docker.

   Rationale for hashing source inputs (not rendered HTML): Next's served HTML embeds per-build
   asset hashes / RSC payloads, so it is not stable across builds. Source-input hashing is
   deterministic and is exactly "regenerate when the resume page changes". (-> learnings)

5. **`package.json`** - `"resume:pdf": "node scripts/generate-resume-pdf.mjs"` and
   `"resume:pdf:check": "node scripts/generate-resume-pdf.mjs --check"`. No new deps.

6. **`next.config.ts`** - set `outputFileTracingRoot: import.meta.dirname` so `output: standalone`
   emits `server.js` at `.next/standalone/server.js` even inside the nested `.worktrees/` checkout
   (learnings 0002); no-op in CI/Docker (single root). Needed because the generator + smoke test
   boot the standalone server.

7. **`public/resume.pdf` + `public/resume.pdf.hash`** - committed tracked artifacts (so Docker /
   runtime serve a static file, no browser). Generated by step 4.

8. **`AGENTS.md`** - prepend a "Public repo - strip PII before commit" section (outside the
   trellis/spectra managed markers).

9. **CI** (`.github/workflows/deploy.yml`) - add `- run: npm run resume:pdf:check` to the verify
   job (cheap, no Chrome).

10. **`tests/smoke.test.mjs`** - assert `GET /resume.pdf` -> `200` + `content-type:
    application/pdf` (the committed PDF is served from `public/`).

11. **Reflect** - `features.md` (`/resume` -> live), `architecture.md` (committed + hash-gated PDF;
    `outputFileTracingRoot`), `learnings.md` (rendered-HTML hash is non-deterministic; CI verifies
    via a hash check, not a browser).

## Verification

- `npm run lint`, `npm run build`, `npm test` green (run from a clean single-root export if the
  worktree nesting interferes, per learnings 0002 - though step 6 should fix that).
- `npm run resume:pdf` produces a clean, chrome-free, light `public/resume.pdf`; re-running is a
  no-op; `--check` passes after commit and fails after an un-regenerated resume edit.
- Manual: `/resume` reads cleanly at 320px and full width, light + dark; the PDF opens and contains
  no phone/email/street/postal.
